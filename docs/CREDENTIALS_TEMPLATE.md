## 引き渡し時の注意事項

### ✅ 実施すべきこと

1. **パスワードの変更**
   - 引き渡し前に、すべてのパスワードを変更することを推奨
   - 特にデータベースパスワードとSSHパスワード

2. **新しいデータベースの作成**
   - 現在は個人用のデータベースを使用しているため、SE部用の新しいデータベースを作成
   - スキーマは以下のSQLで再作成可能:
     ```bash
     # サーバーに接続
     cd ~/www/cgi-bin/requestForm_api
     python3
     >>> from app import db
     >>> db.create_all()
     >>> exit()
     ```

3. **データ移行（必要な場合）**
   - 既存データを移行する場合は、バックアップからリストア
   - 詳細は [運用マニュアル - バックアップとリストア](specs/operations/01_operations_manual.md#4-バックアップとリストア) を参照

4. **環境変数の更新**
   - `.env_for_mySQL` ファイルを新しい認証情報で更新
   - サーバーの `.env` ファイルを更新

5. **GitHubアクセス権の設定**
   - SE部メンバーをリポジトリのCollaboratorとして追加
   - または、組織アカウントへリポジトリを移管

## セキュリティベストプラクティス

### パスワード管理

- **パスワードマネージャーの使用**: 1Password、Bitwarden、LastPassなど
- **暗号化されたファイル**: GPG、VeraCryptなどで暗号化
- **定期的な変更**: 3ヶ月ごとにパスワードを変更

### 共有方法

- **推奨**: パスワードマネージャーの共有機能
- **非推奨**: メール、Slack、平文ファイル

### アクセス制御

- **最小権限の原則**: 必要な人にのみアクセス権を付与
- **アクセスログの監視**: 定期的にログを確認
